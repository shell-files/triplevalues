### 🍀 ***TripleValues***
#### 1. 핵심 심볼 : 세 개의 삼각형
삼각형 세 개가 서로 맞물린 구조로, 세 팀원이 각각 E(환경) · S(사회) · G(지배구조)를 대표합니다. </br>
삼각형은 안정성과 방향성을 동시에 상징하며, 겹치는 영역은 세 가치가 분리되지 않고 유기적으로 연결된다는 의미를 담았습니다.

#### 2. 색상 의미
- **Forest Green(`#2d7a4f`)** : Environment (환경, 자연)
- **Ocean Blue(`#1a5fa8`)** : Social / Governance (신뢰, 투명성)
- **Amber(`#f5c842`)** : 중심 포인트 → 세 가치가 만나는 핵심 가치 (Value)

## ✅ 주요 기능 (업데이트 중)

#### 1. 원청사용
| 매뉴명 | 상위 메뉴 | 진행 업무 내용 |
| --- | --- | --- |
| 대시보드 | 기준 및 파트너 정보 | ESG 알루미늄 공급망 관련 대시보드를 확인 할 수 있는 화면 |
| 협력사 정보 | 기준 및 파트너 정보 | 등록된 협력사 정보를 확인 할 수 있는 화면 |
| BOM 관리 | 기준 및 파트너 정보 | 원청사에서 사용한 제품을 제품군별로 자재 데이터를 확인 혹은 등록 하는 화면 |
| PO 관리 | 구매 및 자재 관리 | 원청사에서 납품 받은 송장 번호를 받아오는 화면 |
| 원자재 관리 | 구매 및 자재 관리 | PO 관리의 송장 번호를 기준으로 원자재 데이터를 요청 및 승인 하는 화면 |
| 리스크 현황 | 실사 및 평가 | 각 협력사에서 발생되는 위험별 지표별 리스크 현황을 보여주는 화면 |
| 현장 실사 | 실사 및 평가 | 협력사에서 문제가 생겼을때 현장 실사 나가 작성한 자료를 보고서로 작성하는 화면 |
| 자가진단 보고서 | 실사 및 평가 | 협력사등록 시 최초 작성한 자가진단 보고서를 볼 수 있는 화면 |

#### 2. 협력사용
| 매뉴명 | 상위 메뉴 | 진행 업무 내용 |
| --- | --- | --- |
| 원자재 등록 | 구매 및 자재 관리 | PO 관리의 송장 번호를 기준으로 원자재 데이터를 저장, 수정, 요청, 승인 하는 화면 |

#### 3. 명명 및 설계 규칙
| 구분 | 명명 및 설계 규칙 | 비고 |
| --- | --- | --- |
| DB 통합 | - 컬럼: 소문자 스네이크(snake_case) + 코멘트 필수</br>- 테이블: 대문자 조합(UPPER_CASE)</br>- 조합식: Unicode (한국어 지원) | 데이터 정의 표준화 |
| Backend | - 변수/함수: 카멜 케이스(camelCase)</br>- 모듈화: 기능별 모듈화 (Class 사용 지양)</br>- 파일: 소문자 단수 / 폴더: 소문자 복수형</br>- API: Restful 형식 (5가지 Method 준수) | 기능 중심 개발 |
| Frontend | - 변수/함수: 카멜 케이스(camelCase)</br>- 파일: 파스칼 단수 / 폴더: 소문자 복수형</br>- 컴포넌트: 최소 2회 이상 재사용 시에만 생성 | 효율적 UI 관리 |
